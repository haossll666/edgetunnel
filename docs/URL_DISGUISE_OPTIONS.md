# 伪装 URL（`URL` / 反代页）候选与挑选原则

本文档对应需求梳理 **§3.A2a**：不修改代码默认值，仅给出可操作的候选集合与风险说明，便于在 `/admin` 中自行替换 `config` 里的伪装页地址。

## 挑选原则（优先顺序）

1. **HTTPS 可握手**：页面由主流 CDN/托管提供，TLS 证书链常见、不易被本地环境单独拉黑。
2. **体积与跳转少**：首屏资源适中，避免 Worker 反代分支消耗过多外部 subrequest（免费层 50/invocation）。
3. **大陆可达性波动可接受**：大站也可能被区域性干扰；无「永久安全」的单一 URL，需按需更换并观察。
4. **与代理场景兼容**：避免强 CSP / 强反爬导致反代 HTML 异常（若出现异常，换候选）。

## 候选列表（示例）

以下均为**公共站点首页或常见入口**，仅作「看起来像正常浏览」的参考，**不保证**长期可用或符合当地法规；请自行评估。

| 候选 | 示例 host（仅作占位） | 适用性说明 | 主要风险 |
|------|----------------------|------------|----------|
| GitHub | `https://github.com` | 开发者流量大，TLS 指纹常见 | 部分地区对 GitHub 访问不稳定；政策与合规需自负 |
| Wikipedia | `https://www.wikipedia.org` | 百科类、页面结构稳定 | 部分地区间歇性干扰；语言重定向可能多一跳 |
| Bing | `https://www.bing.com` | 搜索入口，握手模式常见 | 企业/运营商策略可能拦截搜索域 |
| Cloudflare 状态页 | `https://www.cloudflarestatus.com` | 与 CF 生态一致，证书链清晰 | 特征上易与「在用 CF」关联（对个人场景通常可接受） |
| Mozilla | `https://www.mozilla.org` | 非营利站点，页面相对简洁 | 体积与资源可能随改版变化 |
| 维基媒体基金会页 | `https://wikimediafoundation.org` | 与维基生态相关，风格正式 | 同 Wikipedia，存在地域性波动 |
| Internet Archive | `https://archive.org` | 文化类大站 | 部分地区访问质量不一 |
| Fastly 状态页 | `https://status.fastly.com` | CDN 状态类页面，结构简单 | 与「在用边缘/CDN」语义接近，特征性略强 |

> **不要**把本表当作推荐「唯一正确」配置；默认仓库里的占位页（如 `cloudflare-error-page-3th.pages.dev`）若仍可用，可保持不变，仅在不可达或频繁被拦时再切换。

## 操作提示

- 在 `/admin` 中修改伪装 URL 后，用浏览器直接访问你的 Worker 域名下「非业务路径」验证展示是否符合预期（具体路由行为以当前 `_worker.js` 为准）。
- 若出现白屏、无限重定向或大量外链失败，优先换表中**另一类**站点（例如从「搜索类」换到「文档/百科类」），缩小特征碰撞面。

## 与红线的关系

调整伪装 URL **不改变** `/sub?token=...`、`/admin/config.json` 结构、`KEY`/`UUID` 等契约（参见 `AGENTS.md` 与需求文档 §1.1）。
